# Future Adapter UIs

Future UIs may include CLI commands, TUI screens, MCP clients, editor integrations, package APIs, and service agents.

From a product point of view, these interfaces should preserve the same experience contract as Pi:

- compact status first,
- exact context on demand,
- semantic writes through typed capabilities,
- visible validation gates,
- no direct edits to generated graph/index state.

Technical distribution and adapter details belong under system clients.
